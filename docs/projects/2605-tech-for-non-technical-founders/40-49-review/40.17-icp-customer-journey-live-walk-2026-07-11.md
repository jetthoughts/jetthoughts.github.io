## 2. Root cause #1 — two competing "next" systems

Every spine page renders BOTH:
- the **auto strip** (`themes/beaver/layouts/partials/blog/course-prev-next.html`) — dumb-linear from `course_sequence.yaml`, visually a boxed, prominent control;
- the **manual footer** ("Next: / If blocked") — encodes the real branching journey.

Sam agents consistently reported clicking the boxed strip over the prose footer. The strip gives the **wrong single answer exactly at the three highest-stakes points**:

| Page | Footer (correct) | Auto strip (what Sam clicks) | Damage |
|---|---|---|---|
| 2.4 | return to 2.1, RUN the 10 interviews, then 2.5 | Next → 2.5 | arrives at the scoring gate with zero transcripts |
| 2.5 gate | build→2.6 / pivot→2.3 / kill→1.1 | Next → 2.6 only | pivot/kill founders funneled into building |
| 5.1 | core path → 5.3 (5.2 optional) | Next → 5.2 | core reader detoured into optional lesson |

(4.4→4.5 also diverges, but 4.5 is a harmless optional read — low stakes.)

The partial already supports `course_nav: false` per page; the cheap robust fix is per-entry overrides in `course_sequence.yaml` (e.g. `next_override`, `next_label`, or `branch: true` to render the footer's options instead of a single Next).

## 3. Root cause #2 — story mode (walkthroughs) is half-wired

- Chain is inconsistent: M1wt → M2wt (walkthrough-to-walkthrough), then M2/M3/M4wt eject to the spine, M5wt ends nowhere.
- **module-5-walkthrough-mia is a hard dead end** — no strip, no Next, ends into the global site footer. It's linked from every M5 lesson.
- M1 walkthrough: 1,796 words, zero images, no nav strip (forward path buried in last prose paragraph).
- Walkthrough pages (and template pages like vibe-prd-template) lack the nav strip generally — they're not in `course_sequence.yaml`, so the partial skips them.

## 4. Root cause #3 — no graduation

The only "the core course is complete" language lives on **optional 5.7**. A Sam who correctly stops at core-terminal **5.6 gets no completion moment** and is never handed to the landing page's (good, but buried) "Going further" index. Landing has no finisher state at all — it still opens "Start Chapter 1.1" for someone who just completed everything.

## 5. Legacy companion pages that missed the v2 migration (both RED)

1. **build-path-decision-worksheet** — (a) Q1 requires "2-5 pre-orders/paid pilots" as a mandatory gate while 4.1 explicitly says pre-orders come later in M5 and must NOT gate the build decision → false "you're not ready" bounce; (b) routes to **nonexistent modules "6A", "6B", "5.2 Fractional CTO", "Module 2 retake"** — pre-renumber ghosts; (c) printable form keeps raw jargon (WebSockets, row-level security, multi-tenant) that 4.1 translates.
2. **self-serve-stack-walkthrough** — entire frame is "4 weeks, 90 min/day, Mon-Fri" full-time pace, contradicting 4.4's "plan the 10-week version" canon (Mia's M4 walkthrough correctly models ten weeks); Week 3 requires `stripe listen` in a terminal, contradicting 4.3's "without ever opening a terminal."
