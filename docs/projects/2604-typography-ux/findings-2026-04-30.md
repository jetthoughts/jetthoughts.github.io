# Typography & UI/UX Findings — 2026-04-30

Consolidated from 2 specialist agents that completed (typography forensics + reader experience). 3 other agents (competitor benchmarking, mobile UX, cross-page consistency) hit Anthropic rate limits before returning results — re-run after 1:10am Lisbon reset if needed.

Both agents converged on the same #1 issue: **link contrast/affordance fails the WCAG AA bar AND the reader's scan-distance test**. Different methods, same conclusion.

---

## Convergent issues (both agents flagged)

### 1. Link contrast / underline visibility (CRITICAL)
- **Forensics**: `.blog p a { color: #1a8cff }` measures **3.37:1** on white — fails WCAG AA (4.5:1 required for body text).
- **Reader**: 40%-alpha underline at desktop scan speed reads as "blue word" not "clickable source." Most acute on post #5 (`47-startups-failed-same-coding-mistake`) where every cited stat is a link.
- **Why painful**: every citation is a credibility moment. Burned-founder ICP needs to see "this stat is sourced" at a glance. Faint link = lost trust.
- **Fix**: change `#1a8cff` → `#0066d6` (4.78:1). Bump `text-decoration-color` opacity from `rgba(...,0.4)` to `rgba(...,0.7)` or solid. Add `text-decoration-thickness: 2px`. One-block CSS change in `themes/beaver/assets/css/style.css`.

### 2. List items feel disconnected (both agents)
- **Forensics**: mystery 12px top margin on `.blog li` (inherited from a deeper bundle) + 30px bottom = **42px total between items** — looks like paragraphs, not a list.
- **Reader**: "30px margin-bottom feel disconnected — drop to 12-14px so groups read as one unit."
- **Fix**: `.blog li { margin-top: 0; margin-bottom: 0.75em; }` to override the inherited margin. Lists tighten to ~15px gap, still readable.

## Reader-experience (presentation-level, not writing)

### 3. End-of-article CTAs are invisible (HIGHEST CONVERSION IMPACT)
After 11–12 min reading, the conversion mechanism is:
- Post #5: a plain inline blue link inside a paragraph
- Post #6: an italicized line below an `<hr>`
- Post #7: a blockquote that reads as an italic aside, not a callout
- Post #1: nothing at all

ICP-E hits the bottom and visually defaults to "I guess that's it?" Highest-intent moment, weakest visual treatment.

**Fix proposal**: introduce a `.cta-link` button class.
```css
.blog .cta-link {
  display: inline-block;
  background: #cc342d;
  color: #ffffff;
  text-decoration: none;
  padding: 14px 28px;
  border-radius: 8px;
  font-weight: 600;
  font-size: 18px;
}
.blog .cta-link:hover { background: #a8231e; }
```
Use it on the booking link in every post's closing CTA. Authoring change is one wrap per post.

### 4. Lead-magnet blockquote reads as a footnote, not a callout
Italic + 18px + 4px Ruby red border + no background fill = "secondary content" signal. Opposite of what a CTA needs.

**Fix**:
```css
.blog blockquote {
  background: rgba(204, 52, 45, 0.05);
  border-left-width: 6px;
  padding: 24px 28px;
  border-radius: 0 8px 8px 0;
  margin: 48px 0;
}
.blog blockquote p { font-style: normal; font-size: 20px; color: #1a1a2e; }
```
Affects post #7 today, every future lead-magnet hook tomorrow.

## Typography-forensics CSS findings

### 5. H2 + H3 have NO mobile breakpoint
- Mobile H1 drops to 32px, H2 stays at 29px, gap is only 3px → hierarchy collapses.
- **Fix**:
  ```css
  @media (max-width: 768px) {
    .blog h2:not(.post-title) { font-size: 24px; line-height: 32px; margin-top: 40px !important; margin-bottom: 14px !important; }
    .blog h3:not(.post-title) { font-size: 19px; line-height: 26px; margin-top: 32px !important; margin-bottom: 12px !important; }
  }
  ```

### 6. UL/OL containers inherit body's `font-weight: 300`
- LI/blockquote-p/pre-code children correctly override to weight 400, but the **list bullets are sized by the UL** (mismatch with item text), and any direct text node inside blockquote or pre renders thin.
- **Fix**: `.blog ul, .blog ol, .blog blockquote, .blog pre { font-weight: 400; font-size: 20px; line-height: 35px; }`

### 7. H1 spills outside the body column
- H1 renders at 924px wide, body column is 688px → title looks orphaned, body looks indented.
- **Fix (CSS-only)**: `.blog h1, .blog .post-meta-byline, .blog img.post-cover { max-width: 700px; margin-left: auto; margin-right: auto; }`
- Or better (template fix): move H1/byline/cover into the same `.container` wrapper as the post body in `themes/beaver/layouts/blog/single.html`.

### 8. Inline code line-height bug
- `.blog :not(pre) > code` inherits `line-height: 1.75` (unitless) → wrapped inline code's second line offsets from the surrounding 35px paragraph baseline.
- **Fix**: `.blog :not(pre) > code { line-height: inherit; }`

### 9. Cover image not retina
- Rendered 688×361, naturalWidth=688 (1:1, no 2× asset). Lighthouse will flag on high-DPR displays.
- **Fix**: verify the `wsrv.nl <picture>` partial provides a 2× source. If not, update `themes/beaver/layouts/partials/blog/img-cropped.html` to emit `srcset` with both 1× and 2×.

### 10. Byline color borderline + visited link state unstyled
- `.blog .post-meta-byline { color: #6b7280 }` = 4.83:1 (passes by 0.33). Bump to `#5f6470` (~5.6:1).
- No `:visited` style → browser default purple appears after navigation. Add explicit visited color (e.g. `#0a4f9a`).
